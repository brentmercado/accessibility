Web Accessibility

Roughly 15 percent of the world's population lives with some form of disability, of whom 2 to 4 percent experience significant difficulties in functioning. Disabilities come in many forms such as vision and/or hearing impaired, difficulty with motor functions, and attention spans. The purpose of this web page is to demonstrate web accessibility. Websites should be accessible by everyone, and as web developers, it is our responsibility to build experiences for all.

This website can be navigated with only a keyboard by using the Tab button to move forward, and Shift + Tab buttons to move backwards. Users can also experience this website using a screen reader. Some popular choices for screen readers are VoiceOver for Mac users and NVDA for Windows. Color contrast was also taken into account and conforms to WCAG Level AA standards. Users can download the Chrome Extension NoCoffee to view the web page from the perspective of a person suffering from vision disabilities.

Tab Trapping

Tab Trapping forces users to stay in a specific section of a page when navigating around a website. One common use case for tab trapping is when a modal is displayed. Once the modal displays by clicking the Open Modal button, users will be able to tab through the different modal options. When tabbing past the last item inside the modal, the user will be brought right back to the first option inside the modal. This feature makes it very convenient for keyboard only users to navigate around the modal. A user can exit the modal by clicking outside the modal or by pressing the Escape button.

ARIA Live Regions

When interacting with a website, users can change the way a page is displayed. An example might be clicking on an image to bring up more information about it. The new information shows up without the whole page reloading and you can visually see it now on the page. Non visually impaired users can see this change very easily, but to a user suffering from visual disabilities, this change is not brought to their attention. The assistive technology does not pick up on this change in the website and may leave users unaware of the new update. This is where ARIA live regions come in handy. Developers have the option to set a live region to either ‘polite’, ‘assertive’, or the default setting of ‘off’. Assertive settings will have the screen reader immediately read aloud the new change, regardless of whatever the screen reader is currently reading. Polite settings will have the screen reader announce the new changes made on the website only when idle. With your screen reader turned on, try incrementing and decrementing the number below.
